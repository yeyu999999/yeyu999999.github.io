CAPL 常用

Keywords

keywrds 符号 Byte
byte unsigned 1 声明一个字节变量
word unsigned 2 声明两个字节变量
dword unsigned 4 声明四个字节变量
int signed 2 声明两个字节整数
long signed 4 声明四个字节变量
int64 signed 8 .
qword unsigned 8 声明八个字节的变量
char . 1 声明一个字符串
float . 8 声.明一个浮点数，8个字节
double . 8 声明一个浮点数，8个字节

特殊符号

特殊符号 Note
$ In CAPL, you can directly access signal values.
For this purpose, write a dollar sign $ in front of the signal name.
value = $EngineSpeed;
@ You can read and change environment variables without using the functions getValue and putValue.
For this purpose, place an @ in front of the name of the environment variable.
intValue = @EnvLightState;
* (1) 指针变量。例：int * p; p是变量名, int * 是整型的指针类型
(2) 乘法：算数运算符

Structuring Elements (XML)

Structure Element Description
Test Module 测试模块用于指定和验证单个开发步骤，检查原型或实施回归和一致性测试。
它们由许多测试用例组成。
Test Group 测试组专门充当执行树结构的分组元素。它不包含实现逻辑。它可以依次包含其他测试组或元素，例如测试用例。
Test Case 测试用例包含测试执行的实际指令。
CAPL Test Case 用CAPL编写的测试用例可以在XML测试模块中引用。
.NET Test Case 用.net编写的测试用例可以在XML测试模块中引用。
Test Case Template 测试用例模板是通用的测试用例，它定义了测试用例内容并包含了参数占位符。
Test Case Instance 测试用例实例是从测试用例模板派生出来的，并填充了参数值。
Position 在索引位置搜索 Structuring Elements (XML)

CAPL Functions

General CAPL Functions

==================== Database Access ====================
lookupMessage：Searches for a message definition in the database(s).
lookupSignal：Searches for a signal in the database(s).
lookupServiceSignal：Searches for a SOME/IP Service Signal in the database(s).
lookupSysvarSearches for a system variable definition.

数据库访问：DBC的报文，SOME/IP的服务信号，系统变量等等

==================== Flow Control ====================
canOffline：Cuts the connection between the node and the bus.
canOnline：Restores the connection of the node to the bus.
stop：Programmed interrupt of the ongoing measurement.

流程控制：切断节点，接通节点，停止测量程序等等

==================== Language Support and Debugging ====================
elCount ：Determines the number of elements of an array.

语言支持和调试：数组元素数量，CAPL文件名称，debug，write输出优先级等等

==================== logging Functions ====================
StartLogging：Starts all Logging Blocks immediately bypassing all logging trigger settings.
StopLogging：Stops all Logging Blocks immediately bypassing all logging trigger settings.

日志记录功能函数

==================== User Interactions ====================
write：Outputs a text message to the Write Window.

用户交互：Write窗户的函数最多

CAN CAPL Functions

==================== Selectors ====================
Message：Detailed description of the CAN message selectors.
ErrorFrame：Detailed description of the CAN Error Frame selectors.

这两个和CAPL事件程序中的this关键字相关。
举例（1）：this.ID == 0x600;
举例（2）：this.Byte(0) == 22;

==================== Event Procedures ====================
on message：Receipt of a CAN message.
on signal：Is called with every signal change.
on busOff：CAN controller goes to Bus Off.

CAPL的事件程序，和 General Event Procedures这个章节存在关联

==================== CANdb API ====================
getMessageAttrInt：Gets the value of a message attribute from the database.
GetMessageID：Finds out the message ID.
setSignalStartValues：Sets the values of the signals in the parameter to the start values defined in the database.

和DBC相关的函数

Diagnostics CAPL Functions

==================== Event Procedures ====================
on diagRequest：Is called when a diagnostic request is received in the ECU simulation.
on diagRequestSent：Is called when a diagnostic request has been sent by the tester.
on diagResponse：Is called when a diagnostic response is received in the tester.

诊断类型的CAPL事件程序

==================== Security Access ====================
diagGenerateKeyFromSeed：Creates a key to execute secured diagnostic functions within devices.
diagSetCurrentSession：Sets the diagnostics session currently active in the ECU.
diagStartGenerateKeyFromSeed：Starts generation of the security key from the seed using a Seed & Key DLL.
_Diag_GenerateKeyResult：Indicates the result of the security key computation started with DiagStartGenerateKeyFromSeed.
diagStartAuth：Starts the diagnostics authentication process.
diagStartAuthGeneric：Starts the diagnostics authentication process generic.
_Diag_AuthResult：Indicates the result of the diagnostics authentication process started with diagStartAuth/diagStartAuthGeneric.

诊断安全访问：Key的创建，生成，计算，验证；设置会话等等

==================== Functions for Access to the Whole Object ====================
diagResize：Adapt size of a diagnostic object to match specified parameter iterations, or set size of bus message to given number of byte.
diagSetPrimitiveData：Sets the raw data of the complete service primitive.

用于访问整个对象的函数。“还没有完全理解透彻”

==================== Functions for Access to One Parameter ====================
diagSetParameter：①Sets the numeric parameter to the specified value. ②Sets a parameter to the symbolically-specified value.

访问一个参数

==================== Functions for Communication and Setup ====================
diagSendNetwork：Sends a Request to all ECUs on this bus.
diagSendRequest：Sends the request object to the ECU.
diagSendResponse：Sends the response object back to the tester.
diagSetTimeout：Specifies the request timeout.

交流和设置的诊断函数。

Object Oriented Programming

Class: DiagRequest：定义诊断请求服务。
Class: DiagResponse：与诊断服务响应的肯定响应。

Test Feature Set

==================== Signal Oriented Tests ====================
checkSignalInRange：根据条件检查信号、系统变量或环境变量的值。
testValidateSignalInRange
testValidateSignalOutsideRange
CheckSignalMatch：将给定的值与信号、系统变量或环境变量的值进行比较
TestValidateSignalMatch
GetRawSignal：获取信号的当前原始值。
getSignal：获取一个信号的值。
SetRawSignal：设置一个信号的原始值。
SetSignal：将传输信号设置为指定的值。
TestResetNodeSignalValues：把一个节点的所有TX信号重置为初始值。
TestResetSignalValue：重置一个信号的初始值。
TestResetSysVarValue：重置一个系统变量的初始值。

针对信号值测试的CPAL函数

==================== **Test Controlling ** ====================
TestSetEcuOffline：从总线断开ECU，由ECU转发的消息将不会转发到总线。
TestSetEcuOnline,：将ECU连接到总线上，由ECU发送的消息将转发到总线上。

==================== Test Report ====================
TestCaseComment：在测试用例中添加一条注释，可以带入报告中，不影响测试结果。
testGetCurrentTestCaseTitle：返回当前测试用例的名称。
testGetTestConfigurationName：返回测试配置的名称。
testGetTestUnitName：返回测试单元的名称。
TestReportFileName：设置报告文件的名称和路径。
TestReportWriteDiagObject：将测试报告中诊断请求的文本解释写入HTML表中。
TestReportWriteDiagResponse：将测试报告中诊断响应的文本解释写入HTML表中。

TestCase注释，TestCase添加外部引用，TestCase诊断等等为主的三个部分

==================== Test Structuring ====================
TestCaseDescription：可以在test case 的报告中写入测试说明。
TestCaseSkipped：将测试用例标记为未执行。只能在MainTest函数中执行。
TestCaseTitle：测试用例的标题是从CAPL程序中的测试用例名称中自动获取的。
TestStep：报告一个不影响结果的测试步骤。
TestStepFail：描述导致错误的测试步骤。
TestStepPass：报告按照预期执行的测试步骤。
TestStepWarning：描述test case 执行时没有错误，但是结果可能会在以后造成问题。

可以testCase里面的函数，用来描述测试测试步骤，测试结果

==================== Wait Instructions ====================
TestAddTriggerTesterAction：为TestCreateTesterAction创建的测试操作添加一个trigger。
TestCreateTesterAction：创建一个测试操作。每当trigger发生时，都会检查该条件。
TestWaitForSignalInRange：检查信号、系统或环境变量的值是否在定义值的范围内。
TestWaitForSignalOutsideRange：检查信号、系统或环境变量的值是否在定义值的范围之外。
TestWaitForSignalMatch：检查得到的值是否与信号、系统变量或环境变量的值相匹配。
TestWaitForTesterConfirmation：创建一个弹出窗口并等待tester确认。
TestWaitForTimeout：等待直到指定的时间结束（延时等待）。
TestWaitForMessage：等待指定的Message出现。
TestWaitForEthernetLinkStatus：Waits for an Ethernet link status.
